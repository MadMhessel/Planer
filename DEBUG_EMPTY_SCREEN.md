# Диагностика пустого экрана

## Что было исправлено:

1. ✅ Добавлен **Error Boundary** - перехватывает ошибки React и показывает понятное сообщение
2. ✅ Улучшена обработка ошибок в **AuthService** - ошибки не блокируют UI
3. ✅ Безопасная инициализация **Firebase** - ошибки конфигурации логируются
4. ✅ Глобальная обработка необработанных ошибок

## Как диагностировать проблему:

### 1. Откройте консоль браузера (F12)

Проверьте наличие ошибок:
- Красные сообщения об ошибках
- Предупреждения (желтые)

### 2. Проверьте Firebase конфигурацию

В консоли должно быть:
- ✅ `Firebase initialized successfully` - все хорошо
- ❌ `Firebase configuration is missing!` - проблема с переменными окружения

### 3. Проверьте переменные окружения

Убедитесь, что при сборке переданы все переменные:
```bash
VITE_FIREBASE_API_KEY
VITE_FIREBASE_AUTH_DOMAIN
VITE_FIREBASE_PROJECT_ID
VITE_FIREBASE_STORAGE_BUCKET
VITE_FIREBASE_MESSAGING_SENDER_ID
VITE_FIREBASE_APP_ID
```

### 4. Проверьте сетевые запросы

Откройте вкладку **Network** в DevTools:
- Проверьте, загружаются ли файлы (index.html, main.js, index.css)
- Проверьте статус ответов (200, 404, 500)

### 5. Проверьте, что приложение собрано

Убедитесь, что:
- Выполнена команда `npm run build`
- Папка `dist/` содержит файлы
- `dist/index.html` существует

## Типичные проблемы:

### Проблема: Белый экран, ошибка в консоли о Firebase

**Решение:**
- Проверьте переменные окружения при сборке
- Убедитесь, что все `VITE_FIREBASE_*` переменные установлены

### Проблема: Белый экран, ошибка "Cannot read property of undefined"

**Решение:**
- Откройте консоль и найдите точную строку с ошибкой
- Проверьте, что все компоненты правильно импортированы

### Проблема: Белый экран, нет ошибок в консоли

**Решение:**
- Проверьте, что `index.html` загружается
- Проверьте, что `main.js` загружается
- Проверьте вкладку **Network** на наличие 404 ошибок

### Проблема: Показывается Error Boundary с сообщением об ошибке

**Решение:**
- Нажмите "Обновить страницу"
- Проверьте детали ошибки в сообщении
- Проверьте консоль для полного stack trace

## Быстрая проверка:

1. Откройте DevTools (F12)
2. Перейдите на вкладку **Console**
3. Обновите страницу (Ctrl+R или F5)
4. Проверьте сообщения:
   - ✅ `Firebase initialized successfully` - хорошо
   - ❌ Любые красные ошибки - проблема

## Если проблема сохраняется:

1. Очистите кеш браузера (Ctrl+Shift+Delete)
2. Попробуйте в режиме инкогнито
3. Проверьте, что сервер запущен и доступен
4. Проверьте логи сервера (если используете Cloud Run)

## Локальная разработка:

Если проблема только в продакшене:
```bash
# Проверьте локально
npm run dev

# Проверьте сборку
npm run build
npm run preview
```
